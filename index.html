The LookML Foundation: Analytics as Code

What is LookML?

LookML is the heart of Looker. It's not a visualization language, but a **modeling language** used to define business logic, metrics, and data relationships. It creates a reusable semantic layer that acts as a single source of truth for the entire organization.

measure: total_revenue {

type: sum

sql: ${TABLE}.sale_price ;;

description: "Total revenue from all sales."

}

By defining metrics once in code, Looker ensures everyone, from the CEO to a data analyst, uses the exact same calculation, eliminating data discrepancies.

The Abstraction Layer

LookML separates complex SQL logic from the end-user, providing a simple, governed interface for data exploration.

Architectural Showdown: LookML vs. DAX

Primary Paradigm Comparison

Looker and Power BI approach analytics from fundamentally different architectural philosophies. LookML generates SQL to be run on the database, while DAX is a formula language that primarily calculates over imported data.

This chart illustrates the core difference: Looker's logic lives in the database via SQL generation, while DAX's logic is primarily executed on an in-memory data model.

Governance & Reusability

A key differentiator is how business logic is governed. LookML's centralized, version-controlled model promotes a single source of truth, while DAX's power is often contained within individual reports, requiring stricter process governance.

Aspect LookML (Looker) DAX (Power BI)
Execution In-Database (Live) In-Memory (Import)
Governance Centralized Code Decentralized Logic
Reusability High (Universal Layer) Moderate (Ecosystem)

A Multi-Layered Security Framework

Three Levels of Control

Looker's security model ensures users only see and do what they are permitted to, controlling access to content, data, and features.

1. Feature Access (Permission Sets)

What can a user DO? (e.g., Develop LookML, Download Data, Administer)

2. Data Access (Model Sets)

What data can a user SEE? (e.g., Access to Sales Model vs. Finance Model)

3. Content Access (Folders)

Which dashboards can a user VIEW or EDIT?

Granular Data Security

Beyond high-level access, Looker provides powerful tools for securing data at the row and column level directly within the LookML model.

üìè

Row-Level Security (RLS)

Uses `access_filter` to dynamically add `WHERE` clauses to queries based on user attributes. A sales manager for Europe only sees European data.

üëÅÔ∏è

Object-Level Security (OLS)

Uses `access_grant` to hide specific fields (columns), joins, or entire Explores from users without the proper permissions, perfect for PII data.
